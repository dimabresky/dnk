# Компонент `dnk:header.promo.bar`

Узкая полоска промо в шапке сайта (~45px по высоте), данные из инфоблока: текст, фон/цвет текста, ссылка, таймер обратного отсчёта, закрытие (localStorage), отдельные изображения для десктопа и мобильной версии. Текст и таймер при наличии картинки выводятся **поверх** изображения.

## Инфоблок

- Рекомендуемый **символьный код** типа/инфоблока: `dnk_header_promo` (как в проекте).
- ID инфоблока задаётся в [`local/php_interface/include/constants.php`](../../../php_interface/include/constants.php): константа **`DNK_HEADER_PROMO_IBLOCK_ID`** (при смене инфоблока обновите значение).

### Поля элемента (стандартные Bitrix)

| Поле | Назначение |
|------|------------|
| **Название** | Служебное имя кампании; также подставляется в `alt` изображения, если свойство `IMAGE_ALT` пустое |
| **Анонс** (`PREVIEW_TEXT`) | Текст в полоске. Тип текста: обычный или HTML (если включён HTML у анонса в настройках инфоблока) |
| **Активность**, **Сортировка** | Показ и приоритет при нескольких подходящих элементах |
| **Даты активности** | Период показа (учитывается вместе с `ACTIVE_DATE`) |

Выводится **один** элемент: сортировка `SORT` по возрастанию, затем по `ID` по убыванию; берётся первый активный с учётом дат.

### Свойства элементов (коды свойств)

| Код | Тип | Описание |
|-----|-----|----------|
| `LINK` | Строка | URL перехода (вся кликабельная область контента внутри полоски) |
| `LINK_TARGET` | Список | `_self` / `_blank` |
| `BG_COLOR` | Строка | Фон полоски, формат `#RRGGBB` |
| `TEXT_COLOR` | Строка | Цвет текста, таймера и крестика, формат `#RRGGBB` |
| `SHOW_TIMER` | Да/Нет (список, XML `Y`/`N`) | Показывать таймер |
| `TIMER_END` | Дата/время (пользовательский тип «Дата/время») | Момент окончания отсчёта |
| `TIMER_LABEL` | Строка | Подпись перед цифрами таймера |
| `ALLOW_DISMISS` | Да/Нет | Показать кнопку закрытия; при закрытии запись в `localStorage` |
| `DISMISS_KEY` | Строка | Ключ для `localStorage` (если пусто — используется `dnk_hp_{ID элемента}`) |
| `IMAGE_DESKTOP` | Файл | Узкое изображение для десктопа |
| `IMAGE_MOBILE` | Файл | Узкое изображение для мобильной версии |
| `IMAGE_ALT` | Строка | Атрибут `alt` у картинок |
| `FONT_SIZE` | Строка | Размер шрифта (например `13px`, `1rem`, `14` — будет `14px`). Пусто — из стилей компонента. Значение проходит санитизацию в `Utils::sanitizeCssFontSize()` |

Если нет ни текста, ни картинок, и нет активного таймера (показ отключён или время уже прошло), элемент не отображается.

### Картинки

- Загружены **оба** файла и URL различаются — используется `<picture>` с переключением по ширине (см. параметр `MOBILE_BREAKPOINT`).
- Загружен **один** файл — он показывается на всех ширинах.
- Изображения режутся по высоте ~45px при выводе (ресайз на стороне сайта).

## Компонент

### Параметры вызова

| Параметр | По умолчанию | Описание |
|----------|----------------|----------|
| `IBLOCK_ID` | из `DNK_HEADER_PROMO_IBLOCK_ID` | ID инфоблока |
| `CACHE_TIME` | `120` | Кеш компонента (сек.), теги сброса по `iblock_id_*` |
| `MOBILE_BREAKPOINT` | `768` | Для `<picture>`: мобильный источник — `max-width: (значение − 1)px` |
| `HIDE_ON_EXPIRE` | `Y` | После окончания таймера скрывать полоску на клиенте |

### Подключение на сайте

В проекте включён вывод через обработчик **`OnEndBufferContent`**: [`HeaderPromoEvents`](../../../php_interface/include/classes/HeaderPromoEvents.php) вставляет разметку компонента сразу после открывающего тега `<body>`. Для админки и ответов без полного HTML вставка не выполняется.

Ручной вызов в шаблоне:

```php
$APPLICATION->IncludeComponent(
    'dnk:header.promo.bar',
    '.default',
    [
        'IBLOCK_ID' => DNK_HEADER_PROMO_IBLOCK_ID,
        'CACHE_TIME' => 120,
        'MOBILE_BREAKPOINT' => 768,
        'HIDE_ON_EXPIRE' => 'Y',
    ],
    false
);
```

### Файлы шаблона

- `templates/.default/template.php` — разметка
- `templates/.default/style.css` — стили
- `templates/.default/script.js` — таймер и закрытие

## Очистка кеша

После изменений в инфоблоке при необходимости сбросьте кеш компонентов/страницы или полагайтесь на короткий `CACHE_TIME` и теги инфоблока.
